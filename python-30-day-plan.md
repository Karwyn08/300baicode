# 30-Day Python Programming Plan

Goal: build strong Python fundamentals, practice every day, and finish with small portfolio-ready projects.

Daily time: 60-90 minutes

Recommended setup:
- Install Python 3.12 or newer.
- Install VS Code or PyCharm.
- Create one folder for daily exercises.
- Use Git if possible to save your progress.

## Week 1: Python Basics

### Day 1: Setup and First Program
- Install Python and a code editor.
- Learn how to run Python files from the terminal.
- Write `hello.py`.
- Practice `print()`, comments, and simple arithmetic.

Exercise:
- Print your name, age, city, and one programming goal.

### Day 2: Variables and Data Types
- Learn variables.
- Study `int`, `float`, `str`, and `bool`.
- Practice type conversion with `int()`, `float()`, and `str()`.

Exercise:
- Create a program that calculates age from birth year.

### Day 3: Input and Output
- Learn `input()`.
- Format strings with f-strings.
- Combine user input with calculations.

Exercise:
- Build a simple BMI calculator.

### Day 4: Operators
- Learn arithmetic, comparison, and logical operators.
- Practice `and`, `or`, and `not`.

Exercise:
- Create a program that checks if a number is positive, negative, or zero.

### Day 5: Conditional Statements
- Learn `if`, `elif`, and `else`.
- Practice nested conditions.

Exercise:
- Build a grade calculator.

### Day 6: Loops
- Learn `for` loops and `while` loops.
- Practice `range()`, `break`, and `continue`.

Exercise:
- Print multiplication tables from 1 to 10.

### Day 7: Week 1 Project
Project: Number Guessing Game
- Generate a secret number.
- Ask the user to guess.
- Give hints: too high or too low.
- Count attempts.

Review:
- Variables
- Input/output
- Conditions
- Loops

## Week 2: Data Structures and Functions

### Day 8: Lists
- Learn list creation, indexing, slicing, and methods.
- Practice `append()`, `remove()`, `sort()`, and `len()`.

Exercise:
- Create a shopping list manager.

### Day 9: Tuples and Sets
- Learn when to use tuples.
- Learn sets and unique values.
- Practice set operations.

Exercise:
- Remove duplicate names from a list.

### Day 10: Dictionaries
- Learn key-value pairs.
- Practice adding, updating, and deleting dictionary items.

Exercise:
- Create a student record dictionary.

### Day 11: Functions
- Learn function definitions, parameters, and return values.
- Practice reusable code.

Exercise:
- Write functions for add, subtract, multiply, and divide.

### Day 12: Function Practice
- Learn default arguments.
- Learn keyword arguments.
- Practice simple validation.

Exercise:
- Create a function that calculates total price after tax and discount.

### Day 13: String Methods
- Learn common string methods.
- Practice `lower()`, `upper()`, `strip()`, `replace()`, and `split()`.

Exercise:
- Build a word counter for a sentence.

### Day 14: Week 2 Project
Project: Contact Book
- Store names, phone numbers, and email addresses.
- Add contacts.
- Search contacts.
- Delete contacts.
- Display all contacts.

Review:
- Lists
- Dictionaries
- Functions
- String handling

## Week 3: Files, Errors, and Object-Oriented Programming

### Day 15: File Reading
- Learn `open()`.
- Read text files.
- Use `with open(...)`.

Exercise:
- Read a file and count its lines.

### Day 16: File Writing
- Write text to files.
- Append data to files.
- Save program output.

Exercise:
- Save a daily journal entry to a text file.

### Day 17: Error Handling
- Learn `try`, `except`, `else`, and `finally`.
- Handle invalid input.

Exercise:
- Build a division calculator that handles zero division and invalid numbers.

### Day 18: Modules
- Import built-in modules.
- Practice `random`, `math`, `datetime`, and `os`.

Exercise:
- Create a random password generator.

### Day 19: Introduction to OOP
- Learn classes and objects.
- Understand attributes and methods.

Exercise:
- Create a `Car` class with brand, model, and year.

### Day 20: OOP Practice
- Learn constructors with `__init__`.
- Create multiple objects.
- Add methods that change object state.

Exercise:
- Create a `BankAccount` class with deposit and withdraw methods.

### Day 21: Week 3 Project
Project: To-Do List App
- Add tasks.
- Mark tasks as complete.
- Delete tasks.
- Save tasks to a file.
- Load tasks when the program starts.

Review:
- Files
- Error handling
- Modules
- Classes

## Week 4: Practical Python and Final Projects

### Day 22: Virtual Environments and Packages
- Learn `venv`.
- Learn `pip`.
- Install and import a third-party package.

Exercise:
- Create a virtual environment and install `requests`.

### Day 23: Working with APIs
- Learn basic HTTP requests.
- Use the `requests` package.
- Read JSON responses.

Exercise:
- Fetch and print data from a public API.

### Day 24: JSON
- Learn JSON format.
- Read and write JSON files.
- Convert between Python dictionaries and JSON.

Exercise:
- Save contact book data as JSON.

### Day 25: CSV Files
- Learn the `csv` module.
- Read and write spreadsheet-like data.

Exercise:
- Read a CSV file of expenses and calculate total spending.

### Day 26: List Comprehensions
- Learn concise list creation.
- Practice filtering and transforming data.

Exercise:
- Create a list of even numbers from 1 to 100 using a list comprehension.

### Day 27: Testing Basics
- Learn why tests matter.
- Use simple `assert` statements.
- Optional: try `pytest`.

Exercise:
- Write tests for calculator functions.

### Day 28: Debugging and Code Quality
- Learn how to read error messages.
- Use print debugging or a debugger.
- Improve variable names and function structure.

Exercise:
- Refactor an older exercise to make it cleaner.

### Day 29: Final Project Build
Choose one final project:
- Expense Tracker
- Weather App
- Quiz App
- Password Manager
- Habit Tracker

Requirements:
- Use functions.
- Use files or JSON for saving data.
- Handle invalid input.
- Keep code organized.

### Day 30: Final Project Polish
- Test your final project.
- Fix bugs.
- Add a short README.
- Push the project to GitHub if possible.

Final review:
- Python syntax
- Data structures
- Functions
- Files
- Errors
- Modules
- OOP basics
- Practical project structure
